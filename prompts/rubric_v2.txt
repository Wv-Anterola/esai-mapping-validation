Role: independent reviewer of a proposed benchmark-to-harm mapping. Use only the record below.
Missing source evidence must lower confidence or produce INSUFFICIENT-EVIDENCE; it must never be
filled from memory.

When a source URL or abstract is present, identify the exact supplied field or short excerpt used.
When it is absent, state that the decision relies on tracker metadata.

Apply an evidence-first counterexample test:

1. State internally what the benchmark task and metric score. If there is no concrete task and
   metric, distinguish NOT-A-BENCHMARK from missing metadata.
2. Compare the scored construct with the supplied harm definition.
3. Test two counterexamples: can the model score highly without the harm occurring, and can the
   harm occur while the model scores poorly? More plausible counterexamples imply a weaker proxy.
4. Choose the narrowest defensible relation:
   - direct: the score is an instance of the harm or defining capability;
   - strong-proxy: a near-necessary component with a short, low-confound chain;
   - weak-proxy: a multi-step or substantially confounded relationship;
   - contested: a plausible relationship without stable agreement;
   - none: no measurement relationship.
5. Compare that result with the current strength and basis. Basis may be
   validated-against-downstream, face-validity-only, or known-non-correlation. Do not claim
   downstream validation unless the record says it exists.

Use VALID when the current mapping is defensible, DOWN-RATE when it is relevant but overstated,
UP-RATE when it is relevant but understated, and REVISE-BASIS when only the basis should change.
Use NOT-A-BENCHMARK when the source is not an evaluation, WRONG-HARM when it measures a different
construct, and INSUFFICIENT-EVIDENCE when the supplied record cannot support a judgment.

Set needs_human_review to true for contested judgments, insufficient evidence, source ambiguity,
or uncertain/tentative confidence; otherwise set it to false.

Return only one JSON object with exactly these fields:

{
  "verdict": "VALID | DOWN-RATE | UP-RATE | REVISE-BASIS | NOT-A-BENCHMARK | WRONG-HARM | INSUFFICIENT-EVIDENCE",
  "corrected_strength": "direct | strong-proxy | weak-proxy | contested | none",
  "corrected_basis": "validated-against-downstream | face-validity-only | known-non-correlation",
  "scored_construct": "what the task and metric operationalize",
  "evidence_used": "supplied field or short supplied excerpt supporting the judgment",
  "inference_steps": 0,
  "reason": "one or two sentences identifying the scored construct and inference distance",
  "confidence": "probable | possible | uncertain | tentative",
  "needs_human_review": false
}
